Overview

As a pilot, your job in the app is simple: log when you start and end your work day, then submit the tower data you collected in the field. The app has three tabs to do this.

🌅

Morning

Log Start Day when you arrive at the site

→
🌇

Evening

Log End Day when you finish field work

After logging your day, go to Data Update to mark which towers you captured and uploaded.

Logging In

Open the Pilot App

Go to https://pilot.varunaat.in in your mobile or desktop browser.

Enter your Login ID

Your admin will provide this (e.g. pilot01).

Enter your Password

Click Login. You'll see the three tabs.

⚠️

Only pilot accounts work here. Admin credentials will be rejected. If login fails, contact your admin to check your account status.

Data Update

After field work, record which towers had their data captured (drone flew over and collected), uploaded (data processed and sent to server), or had an issue.

Select Project, Date, and Pilot

Choose the project, the date the work was done, and your pilot name.

Enter Tower Range

Type the "From" and "To" tower numbers you worked on (e.g. From: 45, To: 90). Click Load.

Tick the checkboxes

For each tower, tick:
✅ Capture — data collected for this tower
✅ Upload — data uploaded/processed
⚠️ Issue — tower has access or replacement problem

Add Issue reasons (if any)

A popup appears asking for the reason for each issue tower.

Click Submit

All tower statuses are saved. The admin can see them immediately.

💡

Towers already marked as captured in a previous submission are highlighted in yellow so you don't accidentally double-submit them.

Select All

Use the checkbox in the column header to tick all towers in that column at once. Useful when you captured all towers in a range.

Non-Working Days

If you cannot work due to weather, access issues, or any other reason, log a Non-Working Day instead of Start Day.

Go to Start Day tab

Select your Project and Date

Toggle "Non-Working Day"

The tower number and photo fields disappear.

Enter the Reason

E.g. "Heavy rain", "Access denied by landowner", "Equipment failure".

Click Submit

The non-working day is recorded. The client dashboard will show this date as non-working.

✅

Non-working days are visible to the client in their dashboard under the "Non-Working Days" panel, so they have full transparency.

Tips & FAQ

Can I log data for a past date?

Yes. Change the date field to the correct date. This is useful if you forgot to log the previous day.

What if I select the wrong project?

Contact your admin immediately. They can correct entries in the system.

The app says "Server is warming up"

The backend is starting up (cold start). Wait 30–60 seconds — it retries automatically. Do not close the page.

Can two pilots submit data for the same project on the same day?

Yes. Each pilot logs their own Start/End Day separately. In Data Update, select the correct pilot name so the records are attributed correctly.

What does "Already Captured" (yellow row) mean in Data Update?

That tower was already marked as captured in a previous submission. You can still update it, but the yellow colour is a warning so you don't double-count.
