Overview

The Admin Portal is the control centre of Pratibimb. As an admin you can create and monitor tower inspection projects, manage the pilots who do field work, give clients access to see progress, and perform bulk data updates.

🏗️ Projects Create & track
👷 Pilots Add & manage
🏢 Clients Share access
🗼 Tower Data Bulk update

Logging In

Open the Admin Portal

Go to https://admin.varunaat.in in your browser.

Enter Admin ID

Type your admin login ID (e.g. admin).

Enter Password

Type your admin password and click Login.

You're in

You land on the Projects tab — the main dashboard.

⚠️

Only accounts with the admin role can log in here. Pilot credentials will be rejected.

Managing Clients

Each client company gets their own account with a unique access key that lets them view their project dashboards. Clients never log in with a password — they use the key.

Creating a Client

Click the Clients tab

At the top navigation of the admin panel.

Click "Add Client"

A form appears asking for Name, Email, and Phone.

Fill in the details and Save

An access key is automatically generated for them.

Share the access key

Copy it from the table and send it to the client. They paste it at client.varunaat.in.

Client Table Columns

ColumnMeaning
NameCompany or person name
EmailContact email for the client
PhoneContact phone number
Access KeyUnique key the client uses to log into their dashboard. Click Rotate to generate a new one (invalidates old key).
ActiveToggle to enable/disable client access
💡

You can also set a custom access key if the client prefers a memorable word. Use the "Set Key" action in the table.

Managing Pilots

Pilots are the field workers who fly drones and log tower inspection data. Each pilot gets a login ID and password for the Pilot App.

Creating a Pilot

Click the Pilots tab

Click "Add Pilot"

Fill in: Name, Login ID, Password, Phone number.

Share credentials with the pilot

They use these at pilot.varunaat.in.

Pilot Table Actions

ActionWhat it does
EditChange name, phone, or reset password
DeactivateBlocks pilot from logging in (keeps their historical data)
DeletePermanently removes pilot account

Managing Projects

A project is a tower inspection assignment — it has a client, a set of towers (numbered), a start date, and optionally a KML file that gives towers their GPS coordinates.

Creating a Project

Click "New Project"

On the Projects tab.

Fill in the form

Name — project title (e.g. "Rajkot Line 2")
Client — pick from existing clients
Total Towers — how many towers in the project
Start Date — when field work begins
Require Photo — toggle if pilots must upload a photo for Start/End Day
KML File — optional GPS route file; gives each tower lat/lng coordinates

Click Save

Towers are auto-generated (Tower 1 to N). The client can now see it in their dashboard.

🗺️

A KML file is exported from Google Earth or GIS software. It defines the route and tower positions on the map. Without it, the map in the Client Dashboard won't show tower locations.

Field Logs Table

Shows every Start Day and End Day entry logged by pilots — who did it, when, and which towers they reached.

Actions

ButtonWhat it does
Save ChangesSaves any checkbox changes made in the tower table
Add TowerAdds an extra tower beyond the original count

⚠️

Reset Data is irreversible. It wipes all captured/uploaded status and field logs for the project. Only use this on test projects.

Bulk Data Update

When you need to update many towers at once (e.g. a pilot submitted data on paper and you're entering it), use the Admin Data Update tab.

Select Project and Date

Choose which project and which date this data applies to.

Enter tower range

Enter "From" and "To" tower numbers (max 1000 at once) and click Load.

Check the boxes

Tick Capture, Upload, or Issue for each tower. Use the header checkbox to select all.

Set Issue reason (if any)

For towers marked Issue, a modal will ask for the reason.

Optionally attribute to a Pilot

Select which pilot did this work for proper attribution in reports.

Click Submit

Data is saved and tower status is updated immediately.

Client Access Keys

Access keys are how clients authenticate to their read-only dashboard. There is no password for clients — just the key.

ActionWhen to use
Rotate KeyWhen a client's key is compromised or you want to revoke access temporarily. Old key stops working immediately.
Set Custom KeyWhen the client wants a memorable key (e.g. "RAJKOT2025"). Must be unique across all clients.
Deactivate ClientBlocks all access without deleting the account.

Roles & Permissions

🔴 Admin
  • Create / edit / delete projects
  • Manage clients and access keys
  • Create / edit / delete pilots
  • Bulk update tower data
  • View all field logs
  • Reset project data
  • Upload / sync KML files
🟢 Pilot
  • Log Start Day / End Day
  • Upload field photos
  • Submit tower data updates
  • View own projects only
  • Mark non-working days
🔵 Client
  • View project dashboard
  • See KPI metrics
  • View tower map
  • View field photos
  • See progress charts
